# Plan Tab — Design Spec

> Combines the weekly itinerary and grocery list into a single tab. One workflow: plan meals → shop for ingredients.

---

## Overview

The Plan tab has two modes controlled by a segment control at the top: **This Week** and **Grocery**. The segment control is always visible and sticky below the nav bar. Both views share state — adding recipes to the itinerary auto-populates the grocery list.

---

## Navigation Bar

- **Left**: Screen title "Plan" (Noto Serif, 22pt, bold, #1C1A17)
- **Right**: "Edit" text button (terracotta #9A4100) — navigates to Itinerary Setup wizard
- No back button (it's a root tab screen)

---

## Segment Control (sticky, below nav)

```
┌─────────────────┬─────────────────┐
│   This Week     │    Grocery      │
└─────────────────┴─────────────────┘
```

- Standard iOS segmented control
- Terracotta (#9A4100) active indicator on cream (#FEF9F3) background
- Sticky below nav bar, does not scroll with content
- Tapping switches between the two views with a crossfade (no slide animation)
- Badge on "Grocery" segment showing unchecked item count (e.g. "12")

- Items grouped by category (aisle): PRODUCE, DAIRY, PANTRY, MEAT, SPICES, etc.

  - 16px top padding before each category, 8px after header

  - Checkbox: 24x24pt, terracotta fill when checked, cream border when unchecked, 44x44pt tap target
  - Ingredient name: Inter, 15pt, regular, #1C1A17 (strikethrough + #8A8279 when checked)

  - Swipe left to delete (standard iOS destructive swipe)
- Summary footer (sticky at bottom): "12 items · 5 checked" — Inter, 12pt, regular, #8A8279, centered, 1px top border #E8DFD2

## Interactions & Transitions

| Action | Result |
|---|---|
| Segment control tap | Crossfade between This Week and Grocery views (200ms) |
| "Plan My Week" button | Push Itinerary Setup wizard onto nav stack |
| "Edit" nav button | Push Itinerary Setup wizard (pre-filled with current preferences) |
| Tonight's card tap | Push Recipe Detail |
| "Start Cooking" button | Push Recipe Detail (user taps "Enter Cook Mode" from there) |
| Day row tap | Push Recipe Detail for that day's recipe |
| ↺ swap button | Animated swap — current recipe slides out, new one slides in (300ms) |
| ✕ skip button | Row dims with crossfade (200ms), "Restore" appears |
| "Get All Ingredients" | Success toast + auto-switch segment to Grocery |
| "Scan Ingredients" | Push Kitchen Scanner as modal (slide up, 400ms) |
| Checkbox tap | Instant toggle with haptic (UIImpactFeedbackGenerator .light) |
| Ingredient swipe-left | Standard iOS destructive swipe with "Delete" action |

## Accessibility

- All tap targets 44x44pt minimum
- Segment control supports VoiceOver with "This Week, tab 1 of 2" / "Grocery, tab 2 of 2"
- Checkbox states announced: "Tomatoes, 2, unchecked" / "Cilantro, 1 bunch, checked"
- Recipe source label on grocery items announced after ingredient name
- Dynamic Type supported on all text — test at largest accessibility size
- Reduce Motion: replace all slide/crossfade with instant cuts
- Swipe-to-delete has visible alternative (edit mode or long-press context menu)

---

## Edge Cases

- **Itinerary has recipes but grocery is empty**: User hasn't tapped "Get All Ingredients" yet. Show a hint banner at top of Grocery view: "Tap 'Get All Ingredients' on your weekly plan to populate this list."
- **Recipe removed from itinerary after ingredients added**: Ingredients remain in grocery list (don't auto-delete — user may have already bought them). Source label updates to show "[removed]" in italic.
- **All ingredients checked**: Show a subtle "All done!" message at the bottom of the list with a confetti-style checkmark animation.
- **Same ingredient from multiple recipes**: Combine quantities and show multiple source labels. E.g. "Garlic (5 cloves)" with "Ramen, Mole" as source.
- **No internet**: Itinerary and grocery list work fully offline (AsyncStorage). Only recipe detail images require network.
