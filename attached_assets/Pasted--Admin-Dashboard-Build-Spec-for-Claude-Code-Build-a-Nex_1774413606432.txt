# Admin Dashboard — Build Spec for Claude Code

> Build a Next.js admin dashboard at `apps/admin/` in the Fork & Compass monorepo. Connects to the existing PostgreSQL database. Manages recipes, users, featured content, and stores every field from recipe-api.com permanently.

---

## Project setup

### Location in monorepo

```
apps/
  mobile/              ← Existing Expo/React Native app
  api/                 ← Existing Express 5 API server
  admin/               ← NEW: Next.js admin dashboard
packages/
  db/                  ← Shared Drizzle ORM schema (existing — extend it)
```

### Tech stack

```json
{
  "framework": "Next.js 14+ (App Router)",
  "ui": "shadcn/ui + Tailwind CSS",
  "orm": "Drizzle ORM (shared from @workspace/db)",
  "database": "PostgreSQL (same instance as mobile app)",
  "auth": "NextAuth.js (email magic link or GitHub OAuth)",
  "hosting": "Vercel",
  "package_manager": "pnpm (matches monorepo)"
}
```

### Init commands

```bash
# From monorepo root
cd apps
pnpm create next-app admin --typescript --tailwind --app --src-dir --import-alias "@/*"
cd admin

# Add dependencies
pnpm add drizzle-orm postgres
pnpm add @auth/nextjs @auth/drizzle-adapter
pnpm add -D drizzle-kit

# Add shadcn/ui
pnpm dlx shadcn-ui@latest init
# When prompted: style=default, baseColor=slate, cssVariables=yes

# Add shadcn components we need
pnpm dlx shadcn-ui@latest add table button input select badge dialog sheet tabs card dropdown-menu separator avatar command popover calendar checkbox label textarea switch toast
```

### Workspace config

Add to root `pnpm-workspace.yaml` if not already there:

```yaml
  - 'packages/*'
```

Add to `apps/admin/package.json`:

## Database schema (Drizzle ORM)

Extend `packages/db/src/schema/` with these tables. Some may already exist — add missing fields to existing tables, create new tables for the rest.

### `packages/db/src/schema/recipes.ts`

```typescript
import { pgTable, text, boolean, integer, decimal, timestamp, jsonb } from 'drizzle-orm/pg-core';

// ============================================

// ============================================
export const recipes = pgTable('recipes', {
  // From API
  id: text('id').primaryKey(),                          // UUID from recipe-api.com
  name: text('name').notNull(),
  description: text('description'),
  category: text('category').notNull(),                  // "Dinner", "Beverage", "Dessert", etc.
  cuisine: text('cuisine').notNull(),                    // "Italian", "Japanese", etc.
  difficulty: text('difficulty').notNull(),               // "Easy", "Intermediate", "Advanced"
  tags: jsonb('tags').$type<string[]>().default([]),
  culturalContext: text('cultural_context'),

  // Our metadata
  countryId: text('country_id'),
  regionId: text('region_id'),
  status: text('status').notNull().default('live'),      // "live", "hidden", "draft"
  featured: boolean('featured').notNull().default(false),
  featuredOrder: integer('featured_order'),
  createdAt: timestamp('created_at').defaultNow(),
  updatedAt: timestamp('updated_at').defaultNow(),
});

// ============================================
// RECIPE META (timing, servings)
// ============================================
export const recipeMeta = pgTable('recipe_meta', {
  recipeId: text('recipe_id').primaryKey().references(() => recipes.id, { onDelete: 'cascade' }),
  activeTime: text('active_time'),                       // ISO 8601 "PT20M"
  passiveTime: text('passive_time'),                     // ISO 8601 "PT1H40M"
  totalTime: text('total_time'),                         // ISO 8601 "PT2H"
  overnightRequired: boolean('overnight_required').default(false),
  yields: text('yields'),                                // "4 servings"
  yieldCount: integer('yield_count'),                    // 4
  servingSizeG: integer('serving_size_g'),                // 300
});

// ============================================
// DIETARY FLAGS
// ============================================
export const recipeDietary = pgTable('recipe_dietary', {
  recipeId: text('recipe_id').primaryKey().references(() => recipes.id, { onDelete: 'cascade' }),
  flags: jsonb('flags').$type<string[]>().default([]),              // ["Gluten-Free", "Dairy-Free"]
  notSuitableFor: jsonb('not_suitable_for').$type<string[]>().default([]),
});

// ============================================
// STORAGE & REHEATING
// ============================================
export const recipeStorage = pgTable('recipe_storage', {
  recipeId: text('recipe_id').primaryKey().references(() => recipes.id, { onDelete: 'cascade' }),
  fridgeNotes: text('fridge_notes'),
  fridgeDuration: text('fridge_duration'),                // ISO 8601 "P4D"

  doesNotKeep: boolean('does_not_keep').default(false),
});

// ============================================
// EQUIPMENT
// ============================================
export const recipeEquipment = pgTable('recipe_equipment', {
  id: text('id').primaryKey(),                            // Generated UUID
  recipeId: text('recipe_id').notNull().references(() => recipes.id, { onDelete: 'cascade' }),
  name: text('name').notNull(),
  required: boolean('required').default(true),
  alternative: text('alternative'),
  sortOrder: integer('sort_order').default(0),
});

// ============================================
// INGREDIENT GROUPS
// ============================================
export const recipeIngredientGroups = pgTable('recipe_ingredient_groups', {
  id: text('id').primaryKey(),                            // Generated UUID
  recipeId: text('recipe_id').notNull().references(() => recipes.id, { onDelete: 'cascade' }),

// ============================================
// INDIVIDUAL INGREDIENTS
// ============================================
export const recipeIngredients = pgTable('recipe_ingredients', {
  id: text('id').primaryKey(),                            // ingredient_id from API
  groupId: text('group_id').notNull().references(() => recipeIngredientGroups.id, { onDelete: 'cascade' }),
```

## Recipe seeding script update

Update `scripts/src/upsert-all-recipes.ts` to seed ALL tables from the cached API JSON. Currently it only seeds the core recipes table — it needs to fan out to all 11 recipe tables.

```typescript
import { db } from '@workspace/db';
import { 
  recipes, recipeMeta, recipeDietary, recipeStorage, 
  recipeEquipment, recipeIngredientGroups, recipeIngredients,
  recipeInstructions, recipeTroubleshooting, recipeChefNotes, 
  recipeNutrition 
} from '@workspace/db/schema';
import { v4 as uuid } from 'uuid';

async function seedRecipe(apiRecipe: any) {
  // 1. Core recipe
  await db.insert(recipes).values({
    id: apiRecipe.id,
    name: apiRecipe.name,
    description: apiRecipe.description,
    category: apiRecipe.category,
    cuisine: apiRecipe.cuisine,
    difficulty: apiRecipe.difficulty,
    tags: apiRecipe.tags,
    culturalContext: apiRecipe.cultural_context,
    status: 'live',
  }).onConflictDoUpdate({
    target: recipes.id,
    set: { name: apiRecipe.name, updatedAt: new Date() },
  });

  // 2. Meta
  await db.insert(recipeMeta).values({
    recipeId: apiRecipe.id,
    activeTime: apiRecipe.meta.active_time,
    passiveTime: apiRecipe.meta.passive_time,
    totalTime: apiRecipe.meta.total_time,
    overnightRequired: apiRecipe.meta.overnight_required,
    yields: apiRecipe.meta.yields,
    yieldCount: apiRecipe.meta.yield_count,
    servingSizeG: apiRecipe.meta.serving_size_g,
  }).onConflictDoUpdate({
    target: recipeMeta.recipeId,
    set: { totalTime: apiRecipe.meta.total_time },
  });

  // 3. Dietary
  await db.insert(recipeDietary).values({
    recipeId: apiRecipe.id,
    flags: apiRecipe.dietary.flags,
    notSuitableFor: apiRecipe.dietary.not_suitable_for,
  }).onConflictDoUpdate({
    target: recipeDietary.recipeId,
    set: { flags: apiRecipe.dietary.flags },
  });

  // 4. Storage
  await db.insert(recipeStorage).values({
    recipeId: apiRecipe.id,
    fridgeNotes: apiRecipe.storage?.refrigerator?.notes,
    fridgeDuration: apiRecipe.storage?.refrigerator?.duration,
    freezerNotes: apiRecipe.storage?.freezer?.notes,
    freezerDuration: apiRecipe.storage?.freezer?.duration,
    reheating: apiRecipe.storage?.reheating,
    doesNotKeep: apiRecipe.storage?.does_not_keep || false,
  }).onConflictDoUpdate({
    target: recipeStorage.recipeId,
    set: { reheating: apiRecipe.storage?.reheating },
  });

  // 5. Equipment
  for (let i = 0; i < (apiRecipe.equipment || []).length; i++) {
    const eq = apiRecipe.equipment[i];
    await db.insert(recipeEquipment).values({
      id: uuid(),
      recipeId: apiRecipe.id,
      name: eq.name,
      required: eq.required,
      alternative: eq.alternative,
      sortOrder: i,
    }).onConflictDoNothing();
  }

  // 6. Ingredient groups + ingredients
  for (let gi = 0; gi < (apiRecipe.ingredients || []).length; gi++) {
    const group = apiRecipe.ingredients[gi];
    const groupId = uuid();

    await db.insert(recipeIngredientGroups).values({
      id: groupId,
      recipeId: apiRecipe.id,
      groupName: group.group_name,
      sortOrder: gi,
    }).onConflictDoNothing();

    for (let ii = 0; ii < (group.items || []).length; ii++) {
      const item = group.items[ii];
      await db.insert(recipeIngredients).values({
        id: item.ingredient_id || uuid(),
        groupId: groupId,
        recipeId: apiRecipe.id,
        name: item.name,
        quantity: item.quantity?.toString(),
        unit: item.unit,
        preparation: item.preparation,
        notes: item.notes,
        substitutions: item.substitutions || [],
        nutritionSource: item.nutrition_source,
        sortOrder: ii,
      }).onConflictDoNothing();
    }
  }

  // 7. Instructions
  for (const step of apiRecipe.instructions || []) {
    await db.insert(recipeInstructions).values({
      id: uuid(),
      recipeId: apiRecipe.id,
      stepNumber: step.step_number,
      phase: step.phase,
      text: step.text,
      textFirstSteps: null,   // Populated later by adapt-language script
      textChefsTable: null,   // Populated later by adapt-language script
      action: step.structured?.action,
      tempCelsius: step.structured?.temperature?.celsius,
      tempFahrenheit: step.structured?.temperature?.fahrenheit,
      duration: step.structured?.duration,
      donenessVisual: step.structured?.doneness_cues?.visual,
      donenessTactile: step.structured?.doneness_cues?.tactile,
      tips: step.tips || [],
    }).onConflictDoNothing();
  }

  // 8. Troubleshooting
  for (const ts of apiRecipe.troubleshooting || []) {
    await db.insert(recipeTroubleshooting).values({
      id: uuid(),
      recipeId: apiRecipe.id,
      symptom: ts.symptom,
      likelyCause: ts.likely_cause,
      prevention: ts.prevention,
      fix: ts.fix,
    }).onConflictDoNothing();
  }

  // 9. Chef notes
  for (let i = 0; i < (apiRecipe.chef_notes || []).length; i++) {
    await db.insert(recipeChefNotes).values({
      id: uuid(),
      recipeId: apiRecipe.id,
      note: apiRecipe.chef_notes[i],
      sortOrder: i,
    }).onConflictDoNothing();
  }

  // 10. Nutrition
  const n = apiRecipe.nutrition?.per_serving;
  if (n) {
    await db.insert(recipeNutrition).values({
      recipeId: apiRecipe.id,
      calories: n.calories?.toString(),
      proteinG: n.protein_g?.toString(),
      carbohydratesG: n.carbohydrates_g?.toString(),
      fatG: n.fat_g?.toString(),
      saturatedFatG: n.saturated_fat_g?.toString(),
      transFatG: n.trans_fat_g?.toString(),
      monounsaturatedFatG: n.monounsaturated_fat_g?.toString(),
      polyunsaturatedFatG: n.polyunsaturated_fat_g?.toString(),
      fiberG: n.fiber_g?.toString(),
      sugarG: n.sugar_g?.toString(),
      sodiumMg: n.sodium_mg?.toString(),
      cholesterolMg: n.cholesterol_mg?.toString(),
      potassiumMg: n.potassium_mg?.toString(),
      calciumMg: n.calcium_mg?.toString(),
      ironMg: n.iron_mg?.toString(),
      magnesiumMg: n.magnesium_mg?.toString(),
      phosphorusMg: n.phosphorus_mg?.toString(),
      zincMg: n.zinc_mg?.toString(),
      vitaminAMcg: n.vitamin_a_mcg?.toString(),
      vitaminCMg: n.vitamin_c_mg?.toString(),
      vitaminDMcg: n.vitamin_d_mcg?.toString(),
      vitaminEMg: n.vitamin_e_mg?.toString(),
      vitaminKMcg: n.vitamin_k_mcg?.toString(),
      vitaminB6Mg: n.vitamin_b6_mg?.toString(),
      vitaminB12Mcg: n.vitamin_b12_mcg?.toString(),
      thiaminMg: n.thiamin_mg?.toString(),
      riboflavinMg: n.riboflavin_mg?.toString(),
      niacinMg: n.niacin_mg?.toString(),
      folateMcg: n.folate_mcg?.toString(),
      waterG: n.water_g?.toString(),
      alcoholG: n.alcohol_g?.toString(),
      caffeineMg: n.caffeine_mg?.toString(),
      sources: apiRecipe.nutrition.sources || [],
    }).onConflictDoUpdate({
      target: recipeNutrition.recipeId,
      set: { calories: n.calories?.toString() },
    });
  }
}
```

## Deployment

### Local development

```bash
cd apps/admin
pnpm dev
# Runs at http://localhost:3000
```

The admin dashboard connects to the same PostgreSQL database your API server uses. Same connection string, same data.

### Environment variables

Create `apps/admin/.env.local`:

```env
# Database (same as your API server)
DATABASE_URL=postgresql://username:password@host:5432/forkandcompass

# Auth (NextAuth.js)
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=generate-a-random-secret-here

# Optional: GitHub OAuth (for login)
GITHUB_CLIENT_ID=your-github-oauth-app-id
GITHUB_CLIENT_SECRET=your-github-oauth-app-secret

# Optional: Email magic link (for login)
EMAIL_SERVER=smtp://user:pass@smtp.example.com:587
EMAIL_FROM=admin@forkandcompass.app
```

For production, these go in Vercel's environment variables dashboard — never committed to git.

### Deploy to Vercel

```bash
# Install Vercel CLI if you don't have it
pnpm add -g vercel

# From the admin directory
cd apps/admin
vercel

# Vercel will ask:
# - Link to existing project? → No, create new
# - Project name? → fork-and-compass-admin
# - Framework? → Next.js (auto-detected)
# - Root directory? → apps/admin
# - Build command? → (leave default)
# - Output directory? → (leave default)
```

### Vercel project settings

After first deploy, configure in Vercel dashboard:

| Setting | Value |
|---|---|
| Framework Preset | Next.js |
| Root Directory | `apps/admin` |
| Build Command | `cd ../.. && pnpm install && cd apps/admin && pnpm build` |
| Install Command | `pnpm install` |
| Node.js Version | 20.x |

The build command goes to monorepo root first to install workspace dependencies, then builds the admin app.

### Custom domain setup

**Target URL:** `admin.forkandcompass.app`

#### Step 1: Add domain in Vercel

Vercel Dashboard → fork-and-compass-admin project → Settings → Domains → Add `admin.forkandcompass.app`

#### Step 2: DNS records

At your domain registrar (wherever you registered `forkandcompass.app`), add:

```
Type:  CNAME
Name:  admin
Value: cname.vercel-dns.com
TTL:   3600
```

#### Step 3: SSL

Vercel handles SSL automatically. The `.app` TLD requires HTTPS (which you already know) and Vercel provisions a Let's Encrypt certificate for you. No manual setup.

#### Step 4: Verify

After DNS propagates (usually 5-30 minutes):
- `https://admin.forkandcompass.app` should load the login page
- SSL certificate should show as valid
- All API routes should work

### Multiple environments

| Environment | URL | Database | Purpose |
|---|---|---|---|
| Local dev | `localhost:3000` | Local PostgreSQL or dev DB | Building and testing |
| Preview | `fork-and-compass-admin-*.vercel.app` | Dev/staging DB | Vercel auto-deploys on PR |
| Production | `admin.forkandcompass.app` | Production PostgreSQL | Live admin access |

Vercel automatically creates preview deployments for every git push and PR. Use a separate staging database for preview so you don't accidentally modify production data while testing.

### Protecting the dashboard

The dashboard should NEVER be publicly accessible without auth. Two layers of protection:

**Layer 1: NextAuth.js middleware** (already in the spec)

```typescript
// apps/admin/src/middleware.ts
export { default } from "next-auth/middleware";

export const config = {
  matcher: ["/((?!login|api/auth|_next/static|_next/image|favicon.ico).*)"],
};
```

Every page except `/login` requires authentication.

**Layer 2: Vercel password protection (optional extra)**

Vercel Pro plan includes password protection for deployments. You can enable this as a second gate:
- Vercel Dashboard → Project → Settings → General → Password Protection → Enable
- Set a password that you share with your team

This protects preview deployments too, so nobody stumbles onto a PR deploy.

### Monorepo deployment — making sure Vercel only rebuilds admin when admin changes

Add to `apps/admin/vercel.json`:

```json
{
  "ignoreCommand": "npx turbo-ignore"
}
```

Or if you're not using Turborepo, use Vercel's "Ignored Build Step" setting:
- Vercel Dashboard → Project → Settings → Git → Ignored Build Step
- Set to: `git diff --quiet HEAD^ HEAD -- apps/admin/ packages/db/`

This means Vercel only rebuilds the admin app when files in `apps/admin/` or `packages/db/` change. Changes to `apps/mobile/` won't trigger an admin rebuild.
